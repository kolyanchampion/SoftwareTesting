# АНОТАЦІЯ

Лабораторна робота присвячена вивченню практичних аспектів тестування REST API з використанням інструменту Postman. У роботі розглянуто основи HTTP-протоколу, методи GET, POST, PUT, DELETE та їх застосування для виконання CRUD-операцій. Практична частина виконана на базі JSONPlaceholder API.

# 1. ВСТУП

## 1.1 Мета лабораторної роботи

Метою лабораторної роботи є отримання практичних навичок роботи з REST API:

# 2. ТЕОРЕТИЧНІ ЗАСАДИ

## 2.1 Що таке REST API?

REST (Representational State Transfer) - це архітектурний стиль для побудови розподілених систем. REST API дозволяє різним програмам комунікувати між собою через інтернет.

## 2.2 Основні HTTP-методи

| Метод | Опис | Семантика |
|-------|------|-----------|
| GET | Отримання ресурсу | Read (читання) |
| POST | Створення ресурсу | Create (створення) |
| PUT | Повне оновлення | Update (оновлення) |
| DELETE | Видалення ресурсу | Delete (видалення) |

## 2.3 Основні HTTP-статус коди

| Код | Назва | Опис |
|-----|-------|------|
| 200 | OK | Запит успішний |
| 201 | Created | Ресурс створено |
| 204 | No Content | Запит успішний, тіло порожнє |
| 400 | Bad Request | Помилка у запиті |
| 404 | Not Found | Ресурс не знайдено |

# 3. ПРАКТИЧНА ЧАСТИНА

## 3.1 Завдання 1: GET-запит (отримання списку постів)

**Мета завдання:** Навчитися отримувати дані через API

**Ендпоінт:** GET https://jsonplaceholder.typicode.com/posts

**Кроки виконання:**

1. Відкрити Postman
2. Створити новий запит
3. Вибрати HTTP-метод GET
4. Ввести URL: https://jsonplaceholder.typicode.com/posts
5. Натиснути кнопку "Send"

**Очікуваний результат:**

- Статус-код: 200 OK
- Формат відповіді: JSON
- Відповідь містить масив з 100 об'єктами

## 3.3 Завдання 3: POST-запит (створення нового ресурсу)

**Мета завдання:** Створити новий ресурс через API

**Ендпоінт:** POST https://jsonplaceholder.typicode.com/posts

**Налаштування Body:**
- Тип: raw
- Формат: JSON

**Тіло запиту:**
```json
{
  "title": "Postman API test",
  "body": "This is a test post",
  "userId": 1
}
```

**Кроки виконання:**

1. Створити новий запит
2. Вибрати метод POST
3. Ввести URL
4. У вкладці Body вибрати raw -> JSON
5. Вставити JSON-об'єкт
6. Натиснути "Send"

**Очікуваний результат:**
```json
{
    "title": "Postman API test",
    "body": "This is a test post",
    "userId": 1,
    "id": 101
}
```

## 3.4 Завдання 4: PUT-запит (оновлення ресурсу)

**Мета завдання:** Оновити існуючий ресурс

- Формат: JSON

**Тіло запиту:**
```json

  "id": 1,
  "title": "Updated title",
  "body": "Updated body",
  "userId": 1
}
```

**Кроки виконання:**

1. Створити новий запит
2. Вибрати метод PUT
3. Ввести URL з ID ресурсу
4. Встановити Body як JSON
5. Вставити оновлені дані
6. Натиснути "Send"

**Очікуваний результат:**

```json
{
    "id": 1,
    "title": "Updated title",
    "body": "Updated body",
    "userId": 1
}
```

- Статус-код: 200 OK
- Дані у відповіді оновлені

# 5. ВИСНОВКИ

У результаті виконання лабораторної роботи було вивчено основи тестування REST API та отримано практичні навички роботи з Postman.

Основні результати:

1. **GET-запити** дозволяють отримувати дані з сервера. Успішно отримано список постів (200 OK) та окремий пост за ідентифікатором (200 OK).

2. **POST-запит** призначений для створення нових ресурсів. Виконано створення поста з поверненням статусу 201 Created та унікального ідентифікатора.

3. **PUT-запит** використовується для повного оновлення існуючих ресурсів. Оновлено дані поста з поверненням статусу 200 OK.

4. **DELETE-запит** видаляє ресурси з сервера. Видалення підтверджено статусами 200 OK або 204 No Content.

5. **Postman** виявився зручним та потужним інструментом для тестування API, що дозволяє швидко виконувати HTTP-запити, переглядати відповіді та аналізувати результати.

Отримані навички можуть бути використані для тестування реальних REST API у професійній діяльності.
